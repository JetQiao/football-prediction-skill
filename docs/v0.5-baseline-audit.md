# v0.5 预测准确度基线审计

> 状态：已完成；作为 v0.4 改造前基线保留。
> 审计日期：2026-07-16
> 基线版本：v0.4.0 / `5a6299a`
> 目标：先量化“不准”发生在哪里，再决定模型和数据链路如何重构。

## 1. 结论

当前系统最主要的问题不是某个权重略有偏差，而是“日常预测、历史回测、数据增强、置信度和价值判断”尚未组成同一条可验证的生产管线。

三赛季、五大联赛的现有滚动回测显示：

- 共评估 15 个联赛赛季、3,456 场比赛。
- Dixon-Coles 统计核心在 15/15 组中，Brier 和 Log-loss 都没有击败赔率去水基线。
- 聚合 Brier 为 `0.2001`，赔率基线为 `0.1922`，相对差约 `+4.15%`。
- 聚合 Log-loss 为 `1.0057`，赔率基线为 `0.9690`，相对差约 `+3.79%`。
- 当前价值策略共触发 1,652 次，聚合 ROI 为 `-6.52%`；仅 3/15 组为正。
- 日常流程在没有手工传入 Elo/xG 和外部市场文件时，实际主要输出官方多玩法市场基线，并非独立统计优势。

因此，v0.5 不应先微调 UI 文案或几个固定权重，而应先建立事件时间快照、生产管线回放、模型注册与经验校准，再重做报告。

## 2. 审计方法

### 2.1 自动化检查

- 使用项目 `.venv` 运行全部测试：36 项通过。
- 使用当前 CLI 和当前默认参数运行回测：
  - `min_train=120`
  - `window=600`
  - `refit_every=30`
- 使用 football-data.co.uk 的以下联赛：
  - 英超 `E0`
  - 德甲 `D1`
  - 西甲 `SP1`
  - 意甲 `I1`
  - 法甲 `F1`
- 覆盖赛季：
  - 2023/24
  - 2024/25
  - 2025/26

本次数据仅用于本地审计，没有提交到仓库。

### 2.2 指标解释

- Brier 和 Log-loss 越低越好，能评价完整概率分布，而不只是“猜中没猜中”。
- ROI 只作为策略结果，不作为概率模型唯一验收指标。
- 赔率基线是同一 CSV 中可用的收盘市场赔率去水概率。
- 当前回测只覆盖 `rolling_backtest()` 的 Dixon-Coles 核心，不能代表日常融合管线已经得到验证；这本身就是本次审计发现的问题之一。

| 赛季 | 样本 | 模型 Brier | 赔率 Brier | 模型 Log-loss | 赔率 Log-loss | 价值策略 ROI |
|---|---:|---:|---:|---:|---:|---:|
| 2023/24 | 1,152 | 0.1960 | 0.1893 | 0.9839 | 0.9553 | -4.67% |
| 2024/25 | 1,152 | 0.2003 | 0.1913 | 1.0068 | 0.9660 | -5.12% |
| 2025/26 | 1,152 | 0.2041 | 0.1959 | 1.0264 | 0.9856 | -9.83% |
| **合计** | **3,456** | **0.2001** | **0.1922** | **1.0057** | **0.9690** | **-6.52%** |

补充观察：

- 三个赛季均落后于赔率基线，且 2025/26 的差距进一步扩大。
- 单个联赛赛季偶尔出现正 ROI，但概率评分仍落后于赔率基线，不能据此认定存在稳定优势。
- 命中率聚合约为 51.16%，但三分类命中率无法反映概率是否过度自信，也无法替代校准指标。

## 4. P0 级问题

### P0-1：日常流程没有加载历史拟合模型

`DailyPipeline` 直接创建 `PredictionEngine(self.settings)`，没有注入 `DixonColesModel` 或 `TemperatureCalibrator`。

结果：

- 有 Elo/xG 时，走 `predict_from_features()` 的启发式计算。
- 没有 Elo/xG 时，走固定联赛中性先验。
- 有官方玩法时，最终概率会退化为官方多玩法反推结果。
- README 中“Dixon-Coles 时间衰减拟合”的能力并没有默认进入日常预测。

### P0-2：日常预测和回测不是同一条管线

当前历史回测只运行滚动 Dixon-Coles：

- 不回放官方玩法联合校准。
- 不回放外部市场融合。
- 不回放 Elo/xG 特征。
- 不回放情报结构化信号。
- 不加载温度校准器。
- 不测试生产环境的价值选择逻辑。

因此，目前无法回答“线上报告中的最终概率，在历史上到底表现如何”。

### P0-3：配置存在，但自动数据增强没有接入日常编排

`THE_ODDS_API_KEY`、`API_FOOTBALL_KEY` 和对应 Provider 已存在，但 `DailyPipeline` 没有调用它们。用户即使配置了 Key，也仍需手工生成并传入文件。

这会导致默认体验长期处于：

- 无外部锐角市场。
- 无结构化伤病与阵容。
- 大量联赛无 Elo/xG。
- 低置信、无价值信号或直接复制市场基线。

### P0-4：预测市场和价值比较市场没有形成强约束隔离

要判断竞彩 HAD 是否有价值，至少应区分：

- `reference_market`：用于提升预测的独立市场，例如 Pinnacle 或多公司共识。
- `target_market`：实际要比较价格的竞彩 SP。

如果只有竞彩自身玩法参与概率反推，则只能输出“市场共识”，不能再把同一运营方的价格包装成独立价值优势。

### P0-5：置信度是手工公式，不是经验校准结果

当前置信度来自：

- 最大概率。
- 模型与市场的一致程度。
- 情报完整度。
- 固定常数。

它没有基于历史可靠性曲线、样本量、模型不确定性或分层表现学习。模型与市场“都错但一致”时，也可能抬高置信度。

## 5. P1 级问题

### P1-1：回测存在同日信息边界不严格的风险

回测按日期排序，但没有把同一比赛日作为一个不可拆分的预测批次。CSV 没有可靠开赛时间时，同一天较早行的赛果可能进入同日较晚行的训练窗口。

v0.5 必须按 `as_of` 截点回放；如果只有日期，则训练集必须严格满足 `match_date < prediction_date`。

### P1-2：价值回测和生产价值选择逻辑不一致

生产 `assess_value()` 会遍历主/平/客并选择最大 EV；回测则先选择最高模型概率结果，再检查该结果的 EV。

这意味着当前 ROI 不是生产价值策略的真实历史表现。

### P1-3：特征覆盖窄且没有使用样本不确定性

当前 xG 主要依赖 Understat 覆盖，ClubElo 主要适用于俱乐部：

- 国家队和大量竞彩联赛会缺失。
- 最近 N 场没有显式按比赛日期排序和时间衰减。
- 没有主客场拆分、对手强度修正、休息天数和赛程密度。
- `sample_size` 没有进入收缩或不确定性计算。
- `form_index` 在自动 Provider 中没有真正构建。

### P1-4：情报影响由输入方直接填写

当前 LLM/人工可直接给出 `impact` 和 `outcome`。虽然有范围限制，但仍存在：

- 同一事实重复计权。
- 主观影响大小无法回测。
- 伤病对胜平负、总进球和比分形态的作用不一致。
- 传闻可信度与球员真实战术价值混在一个数字里。

v0.5 应让智能体只抽取事实，概率影响由确定性球员/阵容模型计算。

### P1-5：官方多玩法校准把高度相关市场当作多组约束

HAD、HHAD、比分、总进球和半全场来自同一运营方，彼此高度相关。当前固定玩法权重和简单比例去水没有通过历史数据学习，可能产生“信息看起来更多，实际只是重复市场意见”的问题。

## 6. UI 审计摘要

当前报告具备离线、自包含、键盘页签和移动端适配等优点，但主要问题是：

- 首屏 Hero 占用过多高度，真正需要比较的赛单在首屏下方。
- 大量 8–10px 文本和低对比灰色，阅读成本高。
- 英文装饰标签、霓虹绿和多层边框过多，削弱研究工具的可信感。
- “数据覆盖”“模型置信”“价值信号”同时使用徽标，语义层级不清。
- 卡片过大，几十场比赛时扫描效率低。
- 移动端依赖内部横向滚动，不适合快速逐场浏览。
- 回测图缺少坐标、基准、置信区间和分层解释。
- 界面突出“推荐方向”，但没有把“应当放弃/无独立优势”作为同等重要的一等状态。

完整设计方向见 `docs/v0.5-report-design.md`。

## 7. v0.5 的发布前提

在满足以下条件前，不应继续强化“价值”“高置信”文案：

1. 日常预测和回测复用同一 `PredictionPipeline`。
2. 每个输入都带 `observed_at/as_of`，能证明预测时点可用。
3. 训练、校准和模型版本进入 manifest。
4. 置信度由历史校准和不确定性生成。
5. 预测市场与目标竞彩价格严格分离。
6. 聚合概率指标至少不劣于同截点市场基线。
7. 价值信号只在独立数据、足够样本和分层验证通过时开放。

## 8. v0.5 实施后复测

完成事件时间快照、统一生产/回测管线、市场角色隔离、模型注册表、样本外融合与校准后，再次使用五大联赛三个赛季回放：

| 管线 | 样本 | Brier | 市场 Brier | Log-loss | 市场 Log-loss | 候选 |
|---|---:|---:|---:|---:|---:|---:|
| v0.5 production | 3,436 | 0.1950 | 0.1923 | 0.9818 | 0.9696 | 0 |
| v0.5 independent-value | 3,436 | 0.2002 | 0.1923 | 1.0053 | 0.9696 | 0 |

结论：

- 生产概率较 v0.4 的 `0.2001 / 1.0057` 有实质改善。
- 市场基线仍更优，因此当前模型不会晋级为“已验证超额”。
- 独立价值模式真实输出 0 个候选，证明新门禁会在证据不足时关闭信号。
- 样本数差异来自当前 CSV 可用行和统一事件时间过滤，不应直接与旧审计的 3,456 场逐行等同。
