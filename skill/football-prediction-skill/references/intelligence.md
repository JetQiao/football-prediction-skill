# A 级情报流程

## 来源优先级

1. 赛事、国家队、俱乐部官方公告与主教练发布会。
2. 可靠通讯社、当地长期跟队记者和主流体育媒体。
3. API-Football 等结构化伤病源。
4. 聚合站与社交媒体只作线索，不得单独支撑高影响结论。

## 必查项目

- 官方大名单、预计首发、门将与中轴线可用性。
- 伤病、停赛、轮换、赛程密度和旅行因素。
- 主教练明确表达的战术或人员信息。
- 仅在至少两个可靠来源一致时纳入更衣室或负面传闻。

## intel.json

```json
{
  "matches": [{
    "match_id": "123",
    "completeness": 0.75,
    "missing": ["客队最终首发"],
    "evidences": [{
      "title": "主队官方确认前锋缺阵",
      "url": "https://example.com/official-news",
      "published_at": "2026-06-29T09:00:00+08:00",
      "credibility": 0.95,
      "impact": -0.05,
      "outcome": "home",
      "note": "近 12 场贡献 35% 非点球 xG"
    }]
  }]
}
```

单条 `impact` 必须位于 -0.08~0.08；它是 logit 残差，不是直接加减概率。保守评估不确定消息，重复报道同一事实只记一次。
