## 1. 项目概述

### 1.1 一句话定位
一个**以 AI 智能体技能（Skill）为核心**的竞彩足球智能分析工具：**按日期**自动拉取竞彩在售的全部足球赛事、赔率与彩果，由智能体联网搜集球队 / 球员情报，用"统计模型（脚本）+ 锐角市场 + 情报微调"的可解释方法，为**当日每一场**比赛给出预测、置信度与价值判断；并对大赛提供小组排名与淘汰赛对阵推演。可选附加一个薄前端读取 Skill 输出做可视化浏览。

### 1.2 与同类项目的差异（对标基准）
> 调研了 一批同类开源项目，确立差异化方向。

| 同类做法 | 局限 | 本项目的差异化 |
|----------|------|----------------|
| 锁定单一赛事（世界杯） | 赛程有限、平时不可用 | **按日期覆盖全部竞彩场次**，每天可用 |
| 朴素泊松（Poisson）预测比分 | 对 0-0/1-1 等低比分有系统性偏差 | **Dixon-Coles** 修正 + 时间衰减加权 |
| 黑盒模型 / 仅 LLM 直觉 | 不可解释、不可验证 | **可解释打分** + **回测/校准**度量精度 |
| 只看历史进球 | 噪声大、预测力弱 | 优先用 **xG（预期进球）** 与评分体系 |
| 不与市场比价 | 无法识别"是否值得买" | 以**锐角公司收盘赔率**为锚做价值判断 |

### 1.3 目标用户
- 有一定足球知识、希望用数据辅助决策的竞彩购彩者（每天看竞彩赛单的人）。
- 也兼顾大赛期间关注小组 / 淘汰赛走势的球迷。

### 1.4 核心价值
- **每日可用**：以日期为轴，今天有哪些竞彩场次、各场怎么打，一屏看清。
- **更精准**：用统计学正确的模型 + 市场锚定 + 可度量的回测，而非"拍脑袋专业感"。
- **可解释**：每个预测都能展开看到依据（实力、状态、伤病、赔率、变盘）。
- **会算价值**：不只给"谁赢"，而是给"这一注是否有正期望（value）"。
- **赛制推演**：大赛期间附带小组排名 + 淘汰赛对阵推演（含"算计对阵"博弈）。

### 1.5 非目标（本期不做）
- ❌ 不做最佳第三名（best third place）的晋级与对阵推算（组合算法复杂，明确排除）。
- ❌ 不做真实下注 / 资金代操作，仅提供分析建议。
- ❌ 不承诺准确率或盈利，输出为概率性参考。
- ❌ 不做用户账号体系、社交分享（本期）。
- ❌ 不做滚球 / 走地实时预测（本期只做赛前）。

## 2. 产品形态决策：Skill + 自包含 HTML 报告（关键）

> 本项目最关键的形态决策。结合三大难点（非结构化情报、跨域、全覆盖）综合评估后确定。

### 2.1 为什么不走"纯前端"
纯前端在本需求下要硬扛三座大山，性价比最差：
- **非结构化情报**（伤病/更衣室/球星状态/负面新闻）：浏览器内几乎无法获取与理解。
- **跨域 CORS**：竞彩官网 / 赔率站 / 新闻站大多禁止浏览器跨域请求。
- **几十联赛的抓取适配器**：需自建并长期维护一堆爬虫。

### 2.2 形态对比
| 维度 | Skill 形态（推荐核心） | Web App 形态 |
|------|------------------------|--------------|
| 非结构化情报（伤病/更衣室/球星） | ✅ 天生强项：LLM + 联网搜索 | ❌ 极难，需自建 NLP 或硬接 LLM |
| 跨域 CORS | ✅ 不存在（工具侧抓取） | ❌ 最大痛点，需代理/转发 |
| 多联赛抓取 | ✅ 按需读页面，无需养爬虫 | ❌ 每源一个适配器并维护 |
| 专业可读报告 | ✅ 直接生成 | ⚠️ 需自拼 UI |
| 常驻可浏览界面 | ⚠️ 靠对话，无常驻 UI | ✅ 体验好 |
| 全天 60 场成本/速度 | ⚠️ 深度情报全做会贵 → **强制分层** | ✅ 批量算便宜 |
| 回测/校准面板 | ⚠️ 流程内做较别扭 | ✅ 适合常驻面板 |
| 统计严谨性 | ✅ 脚本计算，不靠语感 | ✅ 同样 |

### 2.3 结论：Skill 核心 + 自包含 HTML 报告（无前端工程）
- **核心 = Skill**：`SKILL.md` + 脚本（拉竞彩赛单/赔率、Dixon-Coles 计算、回测）。智能体编排：
  拉当天赛单 → 全部场次跑脚本算 B 级预测 → 对重点场联网搜情报做 A 级深度分析 → 产出结果。
  一举解决 **CORS、情报采集、专业分析** 三大难题。
- **输出（每次运行产两个文件）**：
  - `prediction_{date}.json`：机器可读，供回测与跨天累积。
  - `report_{date}.html`：**自包含单文件 HTML 报告**——数据以内联 `<script>` JSON 嵌入，原生 JS 渲染卡片/筛选/排序，图表用**内联 SVG**（零依赖、可离线）。双击即开，无需服务器/构建/部署。
- **彻底免前端工程**：不需要 npm / Vite / Vue/React / 开发服务器 / 部署流水线；数据在生成时已"烤进"HTML，打开时不发请求 → **运行时无 CORS**。
- **数据/Key**：智能体在工具侧抓取，免浏览器跨域；付费源/LLM Key 由运行环境保管。

> 📌 这是一份"每日快照报告"模式：每天跑一次 Skill → 一个可离线、可转发、可存档的 HTML。
> 将来若想做实时常驻 App，JSON 已现成可用，零锁定——但当前需求并不需要它。

## 3. 数据来源（Data Sources）

### 3.1 竞彩赛事 / 赔率 / 彩果（结构化，按日期）
| 数据 | 来源候选 | 说明 |
|------|----------|------|
| 竞彩足球**按日期**赛程、在售场次 | 中国体育彩票官网 / 竞彩公开接口 | 主客队、联赛、开赛/停售时间、赛事编号（如周一 001） |
| 竞彩赔率（胜平负、让球胜平负、比分、总进球、半全场） | 同上 | 官方 SP 值及实时赔率 |
| 彩果（开奖结果） | 同上 | 赛后比分、各玩法开奖 |

### 3.2 市场赔率（精准度的关键锚点）
| 数据 | 来源候选 | 用途 |
|------|----------|------|
| **锐角公司赔率**（Pinnacle 等） | 公开赔率 API / 聚合站 | 去水后作为"最准市场概率"先验与价值比价基准 |
| 欧指 / 亚盘**变化过程**（开盘→收盘） | 赔率聚合站 | 变盘方向作为特征（聪明钱 vs 公众盘） |

### 3.3 球队 / 球员资讯（半结构化 / 非结构化）
| 信息项 | 来源候选 |
|--------|----------|
| 官方大名单 | 各赛事 / 俱乐部官网、权威媒体 |
| 球员资料（年龄、身高、位置、所属俱乐部） | 公开足球数据站 |
| 伤病 / 停赛 | 体育新闻、伤病数据站 |
| 更衣室 / 负面新闻 | 体育媒体（需 NLP 情感判断） |
| 球队评分、近 N 场战绩、xG 数据 | 公开数据站 |

### 3.4 数据获取与分层策略
- **Skill 形态优势**：智能体在工具侧抓取，**无浏览器跨域限制**，可读取的来源远多于纯前端方案。
- 结构化（赔率、彩果、评分）：按日期批量拉取并缓存为 JSON（供 F7 回测复用）。
- 非结构化（情报）：**仅对重点场（A 级）联网搜集**（见 §4.3 情报分层），标注来源 + 时间 + 可信度，允许人工覆盖。

### 3.6 推荐采用组合与实时锐角赔率缺口的处理
- **推荐数据栈**：竞彩官方/SportteryAPI（赛单+赔率+彩果+价值）＋ clubelo + understat/soccerdata（评分/xG）＋ API-Football（伤病/阵容）＋ 智能体搜索（软情报）＋ football-data.co.uk（历史回测）。**全部免 Key 或免费注册 Key。**
- **实时锐角赔率缺口的三条退路**（按优先级）：
  1. **付费**：The Odds API / 类似服务的付费档拿实时 Pinnacle 足球（成本不高，最稳）。
  2. **免费验证**：试 OddsPapi 免费档是否真含实时 Pinnacle。
  3. **优雅降级**：拿不到实时锐角时，市场层退化为"竞彩 SP + 亚盘隐含 + 多家欧指共识"，精度略降但模型仍成立；同时用 football-data.co.uk 的 Pinnacle 收盘赔率校准这套退化方案与真锐角的偏差。
> 结论：**回测精度有保障（历史锐角已解决）**；**实时精度取决于是否愿意为锐角赔率付费**，否则走降级方案。这一项需求方需拍板（见 §11 Q3）。

## 4. 功能需求（Functional Requirements）

### 4.1 模块总览（以日期为主轴）
```
┌────────────────────────────────────────────────────────┐
│  竞彩足球智能预测系统                                     │
├────────────────────────────────────────────────────────┤
│  F0 日期赛单（主入口）：选日期 → 当日全部竞彩场次 + 预测卡  │
│  F1 单场详情：赔率/变盘 + 情报 + 预测 + 价值判断           │
│  F2 球队情报中心（分层：重点场完整 / 模型场轻量）           │
│  F3 单场比赛预测引擎                                       │
│  F4 小组赛排名推演         ┐                              │
│  F5 淘汰赛首轮对阵推演(含算计)│ 赛事模式(大赛时激活)         │
│  F6 当日综合建议 + 串关/复式组合 + 分析报告                 │
│  F7 回测与校准面板(精准度度量)                             │
└────────────────────────────────────────────────────────┘
```

### 4.2 F0 — 日期赛单（主入口，新增核心）
- **F0.1** 顶部日期选择器（默认今天），切换日期即加载当日竞彩在售全部足球场次。
- **F0.2** 赛单列表，每行一场，展示：赛事编号、联赛、主客队、开赛/停售时间、胜平负 SP、让球数。
- **F0.3** 每行附**预测卡徽标**：推荐选项（如"主胜"/"让负"）、置信度（高/中/低）、**价值标记**（值得/合理/规避）、情报等级（A/B）。
- **F0.4** 筛选与排序：按联赛、开赛时间、置信度、价值、是否有完整情报筛选；支持"只看高价值/高置信"。
- **F0.5** 关注球队 / 关注联赛置顶。
- **F0.6** 刷新（手动 + 可选轮询），显示数据更新时间与来源；赔率异动提示。

### 4.3 情报分层机制（应对"每日大量场次"）
| 等级 | 触发条件 | 情报深度 | 预测依据 |
|------|----------|----------|----------|
| **A 级 重点场** | 热门联赛 / 高关注 / 用户标记 / 高价值 | 完整 F2（名单、伤病、球星、更衣室、负面 NLP） | 模型 + 市场 + 情报微调 |
| **B 级 模型场** | 冷门 / 信息不足 | 仅评分 + 赔率 | 模型 + 市场 |
- 系统自动分级，用户可手动把任意场升级为 A 级触发深度抓取。

### 4.4 F1 — 单场详情
- **F1.1** 赔率全玩法：胜平负、让球胜平负（含让球数）、比分、总进球、半全场。
- **F1.2** **变盘视图**：欧指/亚盘从开盘到当前的变化曲线与方向标记。
- **F1.3** 双方情报摘要（来自 F2）。
- **F1.4** 预测结果 + 可解释理由清单（来自 F3）。
- **F1.5** 价值判断：模型概率 vs 锐角去水概率 vs 竞彩 SP 概率，三方对比。
- **F1.6** 彩果回填（赛后）与"本场预测是否命中"。

### 4.5 F2 — 球队情报中心（分层）
针对**重点场**的球队，聚合并展示：
- **F2.1 官方大名单**：按位置分组（门将/后卫/中场/前锋）。
- **F2.2 球员状态卡**：年龄、身高、位置、所属俱乐部、号码；伤病（类型 / 影响 / 缺阵概率）；近期表现；可上场概率。
- **F2.3 负面信息聚合**：更衣室矛盾、纪律/场外负面，每条标来源 + 可信度 + 情感倾向（NLP）。
- **F2.4 当家球星状态**：识别核心球员并高亮其状态、伤病、近期表现，评估"球星因素"加成/减损。
- **F2.5 球队综合实力**：攻防评分、阵容深度、历史战绩、数据站评分 → 0–100 实力分（优先采用 xG/评分而非原始进球）。
- **F2.6 球队整体状态**：近 N 场战绩、连续性（连胜/连败）、士气趋势 → 状态指数。
- **F2.7 人工校正**：所有自动抓取项可人工编辑覆盖，并标"已人工确认"。

### 4.6 F3 — 单场比赛预测引擎
- **F3.1** 输入：双方实力/状态/球星/伤停（F2）、主客场、市场赔率与变盘（F1）。
- **F3.2** 计算（方法见 §5）：统计模型概率 + 市场概率 → 融合 → 情报微调。
- **F3.3** 输出：
  - 胜 / 平 / 负概率（校准后）。
  - 推荐比分 Top N（Dixon-Coles 比分矩阵）。
  - 让球玩法建议、总进球区间倾向、半全场倾向。
  - **可解释理由清单**（每个因素如何影响结论）。
- **F3.4** 置信度评级 + 信息缺口提示（如"对方名单未公布，置信度下调"）。

### 4.7 F4 — 小组赛排名推演（赛事模式）
- **F4.1** 录入 / 抓取小组赛赛程与已赛结果。
- **F4.2** 对未赛场次调用 F3，蒙特卡洛模拟推演每组最终积分榜。
- **F4.3** 按官方排名规则计算名次（积分 → 净胜球 → 进球 → 相互战绩，规则可配置）。
- **F4.4** 输出每组**第 1、第 2 名**（第 3 名 / 最佳第三名已明确排除）。
- **F4.5** 展示名次的概率分布（某名次有多种可能时）。

### 4.8 F5 — 淘汰赛首轮对阵推演（赛事模式，核心特色）
- **F5.1** 基于赛事固定淘汰赛框架（如 A1 vs B2…，可配置）+ F4 名次推出首轮对阵。
- **F5.2 "算计对阵"博弈建模**（关键需求点）：
  - 识别"出线已锁定"的球队（末轮前已确保晋级）。
  - 评估其**争第一 vs 拿第二**分别进入的淘汰赛分区与潜在对手。
  - 若"拿第二"路径明显更轻松（避强敌 / 避死亡半区），判断该队**有动机不争第一**，据此调整其末轮预测（可能保守/轮换/不全力）。
  - 输出"策略性让位"提示，并修正最终排名与对阵。
- **F5.3** 展示两套结果：**纯实力对阵** vs **考虑算计后对阵**，并解释差异来源。
- **F5.4** 明确标注：本推演不含最佳第三名，按"仅第 1、2 名"的简化赛制处理，并说明与真实赛制的差异。

### 4.9 F6 — 当日综合建议与组合
- **F6.1** 汇总当日所有场次的预测，给出**当日精选**（高置信 + 高价值）。
- **F6.2** **串关/复式组合建议**：在精选基础上给出串关思路与组合风险提示（仅分析，不下注）。
- **F6.3** 价值排序：把当日所有"值得"的场次按期望值排序。
- **F6.4** 一键生成"当日分析报告"视图（可导出图片 / 打印 / 复制）。
- **F6.5** 理性提示与免责声明显著展示。

### 4.10 F7 — 回测与校准面板（精准度度量，新增）
> 这是"做得更精准"能被**验证**的地方，也是与业余项目的根本区别。
- **F7.1** 历史回测：对过去 N 天的预测，自动比对彩果，统计命中率、ROI、按玩法/联赛/置信度分层的表现。
- **F7.2** **校准评估**：Brier score、log-loss、可靠性曲线（预测 70% 胜的那批，实际是否约 70% 胜）。
- **F7.3** **价值策略回测**：仅按"值得"信号模拟投注的累计收益曲线与最大回撤。
- **F7.4** 权重调参：基于回测结果调整 §5 各因素权重并固化。

## 5. 预测方法论（Analysis Methodology）— 精准度核心

### 5.1 三层融合框架
最终概率 = **统计模型概率** ⊕ **市场（锐角）概率** ⊕ **情报微调**，三者加权融合（权重经回测固化）。

- **统计层**：用 Dixon-Coles 模型（见 5.3）算双方进球期望与比分矩阵。
- **市场层**：以锐角公司收盘赔率去水后的概率为强先验（市场是已知最准的基准之一）。
- **情报层**：用 F2 的伤停、球星缺阵、更衣室、状态对统计概率做有界微调（防止情报噪声主导）。

### 5.2 因素权重模型（可解释打分，初始建议值）
| 因素 | 说明 | 初始权重 |
|------|------|----------|
| 市场（锐角）概率 | 收盘赔率去水 | 35% |
| 统计模型（Dixon-Coles） | xG/评分驱动的进球期望 | 25% |
| 综合实力差 | F2.5 实力分之差 | 12% |
| 近期状态 | F2.6 状态指数 | 8% |
| 球星可用性 | 当家球星出战/状态 | 7% |
| 伤停影响 | 关键球员伤病/停赛累计 | 7% |
| 主客场 / 赛事重要性 | 主场优势、赛事级别 | 3% |
| 变盘信号 | 欧指/亚盘开盘→收盘移动 | 2% |
| 负面/更衣室 | NLP 情感修正项 | 1% |
> 权重为初始值，须经 F7 回测调参后固化。

### 5.3 比分模型：Dixon-Coles（相对朴素泊松的升级）
- 在双泊松基础上加入**低比分相关性修正项**（修正 0-0/1-0/0-1/1-1 偏差）。
- **时间衰减加权**：近期比赛权重更高。
- 攻防强度优先用 **xG** 估计（比实际进球更稳）。
- 输出完整比分概率矩阵 → 推导胜平负、让球、大小球、比分 Top N、半全场。

### 5.4 赔率 → 概率（去水/去 vig）
- 对赔率去除返还率后归一化得隐含概率。
- **竞彩 SP 反水高、更新慢**，不作为"真实概率"，仅作为**比价对象**；以锐角赔率为真实概率锚。

### 5.5 价值判断（Value Bet）
- 价值 = 模型概率 − 竞彩 SP 隐含概率。
- 仅当 `模型概率 × 竞彩赔率 > 1 + 安全阈值` 且模型置信度足够时，标记"值得"。
- 避免在市场高效的玩法上盲目下注（控制假阳性）。

### 5.6 置信度与信息完整度
- 关键信息缺失（名单未出、伤病不明）时**下调置信度并显式提示**，杜绝"伪精确"。
- 模型概率与市场概率严重背离时，触发"高风险/高价值"双向提示。

### 5.7 校准（让概率"可信"）
- 用 Platt scaling / isotonic regression 等对输出概率做校准，使"说 70% 就接近 70%"。
- 校准质量由 F7 的可靠性曲线与 Brier score 监控。

## 6. AI / 专业分析能力实现方式（Skill 形态下）

在 Skill 形态里，智能体本身即 LLM，"专业分析"天然由其完成，分工如下：
- **结构化计算交给脚本**（不靠语感）：Dixon-Coles 比分矩阵、市场融合、校准、价值计算、回测，全部在 `scripts/` 内确定性运行，保证可复现、可验证。
- **非结构化判断交给智能体**：对 A 级重点场，联网搜集大名单 / 伤病 / 更衣室 / 负面新闻，做情感与影响评估，并把结论作为**有界微调项**喂回脚本（防止情报噪声主导统计结果）。
- **报告由智能体生成**：把脚本的结构化结果 + 情报判断组织成可读的"当日分析报告"。
- **成本控制**：B 级场只跑脚本（便宜、批量）；A 级深度情报仅对重点场启用。

> 与纯前端方案的关键差异：不再需要"自建 NLP / 用户自备 Key"，情报理解能力是 Skill 自带的。

## 7. 技术架构建议（非约束，供参考）

### 7.1 核心：Skill（智能体技能）
```

  SKILL.md              # 工作流说明：拉赛单→算预测→搜情报→渲染HTML

    backtest.py         # 回测：命中率/ROI/Brier/可靠性曲线
    render_report.py    # 把预测 JSON 渲染成自包含 HTML（内联数据/样式/SVG）
  templates/
    report.html         # HTML 报告模板（内联 CSS + 原生 JS + 内联 SVG 图表）
  data/                 # 缓存 JSON（赛单/赔率历史/情报/预测/彩果）
  out/                  # 产出：prediction_{date}.json + report_{date}.html
```

- **编排**：智能体读 `SKILL.md` 按流程执行；结构化计算交给脚本（保证可复现、统计严谨），非结构化情报由智能体联网搜集与判断。
- **语言**：脚本用 Python（SciPy 做 Dixon-Coles / 泊松、蒙特卡洛）。

### 7.2 输出：自包含 HTML 报告（替代前端工程）
- **单文件**：`report_{date}.html`，数据以内联 `<script type="application/json">` 嵌入。
- **渲染**：原生 JS 生成日期赛单卡片、单场详情展开、按联赛/置信度/价值的筛选排序。
- **图表**：可靠性曲线、收益曲线、比分概率热力图等一律用**内联 SVG**（零依赖、`file://` 可离线打开）。
- **可选汇总**：`index.html` 列出历史各天报告，形成可浏览归档。
- **分发**：双击打开 / 丢任意静态托管（GitHub Pages 等）皆可，无需构建与服务器。

### 7.3 HTML 报告内容结构（交付物规格）
> 这是用户每天实际看到的东西，逐区定义。

1. **报告头**：日期、生成时间、当日场次数、数据来源与免责声明。
2. **当日精选区**：高置信 + 高价值场次置顶（推荐选项、置信度、价值边际、一句话理由）。
3. **赛单卡片列表**（主体）：每场一卡——赛事编号、联赛、主客队、开赛时间、竞彩 SP、让球数、**预测徽标**（推荐/置信度/价值/情报等级 A·B）。支持原生 JS 按联赛/置信度/价值/时间筛选与排序。
4. **单场详情（点击展开）**：
   - 三方概率对比条：模型 vs 锐角去水 vs 竞彩 SP。
   - 比分概率热力图（内联 SVG）、推荐比分 Top N、让球/大小球/半全场倾向。
   - 双方情报摘要（A 级含大名单/伤病/球星/更衣室；B 级仅评分+赔率）。
   - 变盘曲线（内联 SVG）。
   - **可解释理由清单**（每因素如何影响结论）+ 置信度与信息缺口提示。
5. **赛事模式区**（大赛时）：小组积分推演表 + 淘汰赛对阵图（纯实力 vs 算计后两套）。
6. **回测概览区**：截至当前的命中率/ROI/Brier、可靠性曲线、收益曲线（内联 SVG）。
7. **页脚**：理性购彩 / 未成年人禁止购彩声明。

## 9. 数据模型（草案）

```ts
// 比赛（按日期组织）
interface Match {
  id: string;
  date: string;            // 竞彩销售日期 (YYYY-MM-DD)，F0 主轴
  matchNo?: string;        // 竞彩编号，如 "周一001"
  league: string;
  home: TeamRef;
  away: TeamRef;
  kickoffAt: string;       // ISO 开赛时间
  saleCloseAt?: string;    // 停售时间
  odds: Odds;
  oddsHistory?: OddsSnapshot[]; // 变盘过程
  marketOdds?: MarketOdds; // 锐角公司赔率
  result?: MatchResult;    // 彩果（赛后）
  stage: 'league' | 'group' | 'knockout';
  groupId?: string;        // 大赛分组
  intelTier: 'A' | 'B';    // 情报等级
}

interface Odds {
  whdc?: { w: number; d: number; l: number };                  // 胜平负 SP
  handicap?: { line: number; w: number; d: number; l: number };// 让球胜平负
  score?: Record<string, number>;
  totalGoals?: Record<string, number>;
  halfFull?: Record<string, number>;
  updatedAt: string;
}
interface OddsSnapshot { at: string; whdc: { w:number; d:number; l:number }; handicapLine?: number; }
interface MarketOdds { source: string; w: number; d: number; l: number; noVigProbs: { w:number; d:number; l:number }; }

// 球队情报
interface TeamIntel {
  teamId: string; name: string;
  squad: Player[];
  starPlayerId?: string;
  strengthScore: number;   // 0–100
  formIndex: number;
  recentResults: ResultBrief[];
  xgFor?: number; xgAgainst?: number;
  negativeNews: NewsItem[];
  updatedAt: string; manualOverridden: boolean;
}
interface Player {
  id: string; name: string; position: string;
  age: number; height?: number; club?: string; number?: number;
  injury?: { status: 'fit'|'doubt'|'out'; type?: string; impact: number; source?: string };
  formNote?: string; availability: number; // 0–1
}
interface NewsItem { title: string; url: string; publishedAt: string; sentiment: 'neg'|'neu'|'pos'; credibility: number; }

// 预测结果
interface MatchPrediction {
  matchId: string;
  probs: { w: number; d: number; l: number };       // 校准后
  modelProbs: { w:number; d:number; l:number };      // 统计层
  marketProbs?: { w:number; d:number; l:number };    // 市场层
  topScores: { score: string; prob: number }[];
  handicapAdvice?: string; totalGoalsTendency?: string; halfFullTendency?: string;
  reasons: string[];                                  // 可解释理由
  confidence: 'high'|'mid'|'low';
  value?: { pick: string; edge: number; flag: 'value'|'fair'|'risk' };
}

// 回测记录
interface BacktestRecord {
  matchId: string; date: string;
  predicted: MatchPrediction; actual: MatchResult;
  hit: boolean; brier: number; roiIfBet?: number;
}
```

## 12. 成功指标与验收标准

> "更精准"必须可度量，否则是空话。以下为验收门槛（阈值经首轮回测后校准）。

| 维度 | 指标 | 目标 |
|------|------|------|
| **校准** | 可靠性曲线 / Brier score | Brier 优于"全用竞彩 SP 隐含概率"的基线；可靠性曲线接近对角线 |
| **判别** | 胜平负 log-loss | 低于竞彩 SP 基线 |
| **价值** | 仅按"值得"信号模拟投注的 ROI | 回测期 ROI ≥ 0（剔除竞彩返还率后不为负即有意义） |
| **覆盖** | 当日竞彩场次覆盖率 | ≥ 95%（B 级模型场兜底） |
| **可解释** | 每场预测可展开理由 | 100% 有理由清单 |
| **可用性** | 一条命令产出当日 HTML 报告 | 单次运行 < 可接受时长，离线可开 |
| **稳健性** | 单一数据源失效 | 不阻塞整体，降级 + 来源兜底提示 |

> 验收基线 = **竞彩 SP 隐含概率**。模型必须在校准与判别上稳定优于该基线，才算"更精准"成立。

## 14. 术语表

| 术语 | 含义 |
|------|------|
| **竞彩 SP** | 中国体育彩票竞彩的官方固定/参考赔率 |
| **去水 / no-vig** | 从赔率中剔除博彩公司返还率（抽水）后得到的"公平"隐含概率 |
| **锐角公司** | 以 Pinnacle 为代表、低抽水高限额、被认为定价最准的博彩公司 |
| **Dixon-Coles** | 在双泊松基础上加入低比分相关性修正与时间衰减的足球比分模型 |
| **xG（预期进球）** | 根据射门质量估计的进球期望，比实际进球更稳定地反映球队水平 |
| **Elo** | 基于对战结果动态更新的相对实力评分体系 |
| **价值投注 / value bet** | 模型概率 × 赔率 > 1+阈值，即长期正期望的投注 |
| **凯利（Kelly）** | 由胜率与赔率推导的最优下注比例 |
| **Brier score** | 概率预测的均方误差，越低越好，衡量校准+判别 |
| **可靠性曲线** | 预测概率 vs 实际频率的曲线，贴近对角线说明概率校准良好 |
| **校准（calibration）** | 让"说 70% 就接近 70% 真发生"的后处理（Platt/isotonic） |
| **情报分层 A/B** | A=重点场完整情报；B=冷门场仅评分+赔率（见 §4.3） |
| **算计对阵** | 已锁定出线的队为淘汰赛更优路径而策略性不争小组第一（见 §5/F5.2） |
