# Changelog

## 0.5.1 - 2026-07-16

- 将原有单一决策状态拆成“方向判断 + 价值判断”：有可用概率时始终展示主胜/平局/客胜倾向，不再因缺少独立价值验证而整场标记为弃权。
- 新增 `strong / moderate / slight / unavailable` 四档方向状态，以及 `candidate / watch / no_edge / unverified / unavailable` 五档价值状态。
- 目标竞彩多玩法共识现在可以用于方向研究，但继续禁止使用同一价格循环证明价值；兼容保留旧版 `decision_state` 字段。
- 报告重构方向与价值双列、双卡详情、组合筛选和移动端布局，并补充真实 10 场赛单回归：5 场明确、4 场中等、1 场轻微、0 场方向不可用。
- 每日报告协议升级至 `2.1`，manifest 与回测新增方向/价值状态计数；CLI 同步输出方向强度和价值候选数量。

## 0.5.0 - 2026-07-16

- 新增 DuckDB 目录与 Parquet 不可变赛前快照，完整记录 `as_of/observed_at/source_event_id/payload_hash/snapshot_id`。
- 日常预测和历史回放统一使用 `PredictionPipeline`，修复同比赛日赛果泄漏，并强制预测截点早于开赛。
- 明确区分参考市场、目标竞彩与赛后基准市场；目标价格参与概率时自动弃权，禁止循环证明价值。
- 新增本地模型注册表、同日留出的 OOF 训练、Power/Shin 去水比较、正则化多项逻辑融合、温度校准与严格晋级门槛。
- 新增 RPS、ECE、校准斜率/截距、覆盖率、百分比回撤和决策状态评估；未通过门槛的价值策略不再模拟下注。
- 无手工特征时可自动尝试 ClubElo；配置 API-Football 且提供 fixture ID 时自动注入结构化阵容事实。
- 报告重构为高密度概率与价格工作台，新增赛单筛选排序、四类决策状态、右侧详情抽屉、数据血缘和独立回测页面。
- 15 个五大联赛赛季回放中，生产管线聚合 Brier/Log-loss 为 `0.1950/0.9818`，较 v0.4 审计基线改善，但仍未击败市场 `0.1923/0.9696`，因此生产价值候选保持关闭。

## 0.4.0 - 2026-07-03

- 赛单解析改为保留官方返回的全部比赛，支持仅让球在售与全部玩法待开售场次，并增加源数量一致性守卫。
- 新增 HAD、HHAD、比分、总进球、半全场联合校准，生成比赛级一致比分矩阵，不再用固定中性期望进球解释所有比赛。
- 明确区分独立统计模型、多玩法市场基线和数据不足占位，禁止用官方 SP 自我验证价值信号。
- 新增每日赛后结算，输出整体及联赛、置信度、分析模式分层的 Brier、Log-loss、命中率与 ROI。
- 补齐 HHAD-only、玩法待开售、赛后对齐等长期回归测试。

## 0.3.0 - 2026-07-01

- 按足球比赛日产品重新设计自包含 HTML 报告，重构首屏指挥中心、赛程卡和单场详情。
- 新增官方赛程、竞彩 SP、实力特征、外部市场和赛前情报五层来源追踪，明确显示更新时间与降级状态。
- 官方 SP 去水概率加入模型融合；关键实力与市场输入不足时强制低置信，并暂停输出价值信号。
- 修正中文球队名称归一化和 Elo 实力差缩放，新增 119 面可离线内联的国家队旗帜。
- 竞彩官方接口在代理出口受限时自动尝试绕过代理直连，并保留缓存降级能力。

## 0.2.0 - 2026-06-29

- 重构深色 HTML 报告为高密度赛事数据台，新增首屏概览、直观胜平负概率、焦点场次与移动端布局。
- 新增逐场四页签交互：核心结论、五类玩法、比分分布、依据与风险。
- 补齐胜平负、让球胜平负、比分、总进球、半全场五类官方竞彩 SP 的标准化解析。
- 新增半全场 3×3 概率、让球赛果、总进球分布、双方进球、大小球与比分热力图推演。
- 将 EV、Edge 改写为“长期期望”“概率优势”等中文解释，并明确区分官方报价与模型推演。
- 新增玩法解析、概率归一化、报告内容和移动端回归测试。
